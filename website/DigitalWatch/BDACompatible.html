Description
After DigitalWatch 0.701 I added expermental support for BDA cards to DigitalWatch. After doing this I decided that to implement all the existing features of DigitalWatch with BDA i needed to redevelop DigitalWatch from scratch with a different architecture. Since the BDA version of DigitalWatch was missing a few features available in the standard DigitalWatch i decided not to release it officially.

This page lists the different versions and what has been implemented in them.

Martin S has continued on the work by adding support for
- Now & Next (EPG)
- Smooth Font Rendering
- DVR-MS Recording
- Many more BDA compatible cards
For more information see the DigitalWatch 0.7.2.6 Experimental thread.

DigitalWatch 0.724

From this post

Added a DVICO DVBInput mode.

To use with the spectrums THDTV drivers set DVBInput=5 in settings.ini
To use with the DVICO BDA drivers set DVBInput=6 in settings.ini

Now and Next, and Timeshifting are not implemented.

TS Mux Recording is implemented, but in the limited testing i did the resulting file was often corrupt. I use WS for all my recordings so i'm not spending any more time on recording in this version of DW.

In both DVBInput 5 and 6 modules i've re-enabled the reading of signal quality and strength. It seems to be working for the VP, but I'm not sure what it does for the DVICO.

This exe should still work with the twinhan drivers in DVBInput 1, 2, and 4 modes, but i havn't tested it.

For those with a DVICO that have never used DW before you'll need to create yourself a channels.ini file. The scanchannels.exe program that comes with DW0.701 will not work with the DVICO card so you'll have to do it manually. There's information about the file format in the docs, and there are a few threads on here where people have posted example channels.ini files.

DigitalWatch 0.721

From this post

As before this zip file only contains a new DigitalWatch.exe file
To install just replace the DigitalWatch.exe from 0.701 and set DVBInput=5 in settings.ini

Not implemented yet:
- Now and Next
- Timeshifting

Changes:
  • Disabled the signal strength and quality polling. This is only temporary until spectrum resolves any driver issues in this area.
  • Implemented recording for BDA. It relies on the Cyblerlink MPEG Muxer (mpgmux.ax) so you have to have it registered for it to work.
  • Added the ability to pause the recording. (more details in post)
  • Added LoadAudioRenderer function for AudioDecoders.ini (more details in post)
